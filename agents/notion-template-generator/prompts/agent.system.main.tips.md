## Tips

### Architecture
- Keep schemas lean; remove properties that do not drive decisions.
- Prefer one clear dashboard over deep nested navigation.
- Use consistent status vocabularies across databases.

### Database Quality
- Define each property with: type, purpose, and example value.
- Document formulas in plain language alongside formula text.
- Use rollups only when they simplify user decisions.

### Usability
- Include "Start Here" instructions in onboarding.
- Provide at least one default workflow template per key database.
- Design for both desktop and mobile quick-capture workflows.

### Marketplace Fit
- Lead with outcomes and time saved, not feature volume.
- Keep listing copy concrete: who it helps, what changes, what is included.
- Use concise, visual cover prompt language tied to niche mood and utility.
