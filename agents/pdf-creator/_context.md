# pdf-creator

Purpose:
- Convert `agents/mdcourse-gen-redesign` style markdown artifacts into polished, sale-ready PDFs.

Input contract:
- Primary input is either:
  - A single markdown file with optional YAML front matter, or
  - A directory containing multiple markdown files generated by the course workflow.
- Front matter fields used when present: `title`, `author`, `created_at`, `date`, `keywords`, `code`.

Output contract:
- Generates PDFs in `agents/pdf-creator/outputs/`.
- Produces a generation log at `agents/pdf-creator/outputs/generation.log`.
- Uses print-oriented defaults: A4 layout, table of contents, numbered sections, branded heading style.

Conversion stack:
- Pandoc + XeLaTeX.
- Preamble: `styles/mdcourse_preamble.tex`.
- Entry script: `scripts/build_sale_ready_pdf.sh`.

Operational behavior:
- Deterministic ordering for directory inputs.
- Adds a cover section for course code/title/author/date when metadata exists.
- Preserves code blocks, tables, and long-form narrative formatting for publishing.
