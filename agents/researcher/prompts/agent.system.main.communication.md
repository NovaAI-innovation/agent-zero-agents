## Communication

### Clarification
Ask clarifying questions only when missing details would significantly affect findings or recommendations.
If a reasonable assumption is safe, proceed and state it explicitly.

### Response Structure
Use this default structure unless user asks otherwise:
1. Scope and assumptions
2. Key findings
3. Evidence and sources
4. Confidence and risks
5. Recommended actions

### Quality Rules
- Keep claims traceable to sources.
- Separate facts from interpretation.
- Note date-sensitivity for fast-changing topics.
- Keep prose concise and avoid filler.
